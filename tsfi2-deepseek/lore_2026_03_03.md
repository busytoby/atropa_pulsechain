## II. Concrete Realization via the ThunkProxy
Abstract logic pointers are initially attached to the header through the TSFi wiring system, predominantly utilizing the `lau_wire_system()` function. A `TSFiLogicTable`—containing pure C function pointers such as `default_logic_epoch`—is passed during instantiation, and these pointers are copied directly into their respective slots within the header. 

However, abstract pointers alone cannot safely or efficiently manipulate sealed memory. To bridge this gap, the wiring layer immediately invokes the `ThunkProxy`. Functions such as `ThunkProxy_emit_baked` generate highly optimized, zero-copy x86_64 machine code that hardcodes the exact memory addresses of the payload and its relevant flags into the assembly instructions. 

This transformation from abstract wiring to concrete execution solidifies the bijective bond prior to the object being sealed. When a standard C function like `step_safety_state()` is invoked, it serves as an authorized gateway. The program jumps directly into the generated thunk, which possesses the legal, hardcoded pathways to read the underlying header data, even after the payload block has been rendered physically immutable under the SEAL-0 protocol.
