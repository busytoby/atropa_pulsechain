# Insta Software (TOTL) — Technical & Database Analysis

Founded in Santa Ana, California, **Insta Software** (primarily known under the **TOTL** brand, standing for *"Time Out To Learn"*) was a productivity software publisher during the 1980s. Rather than compiling game logic, Insta Software designed tools centered on home business organization, word processing, and time management for the Commodore 64 and VIC-20.

This document reviews their technical database schemas, the productivity engine, and schedules a priority task layout in Yul mimicking their time management logic.

---

## 1. Flagship Productivity Line

Insta Software marketed their products under the **TOTL** suffix, targeting structured data formats:
*   **TOTL.INFOMASTER**: A relational-like database manager allowing custom text field parsing.
*   **TOTL.TIMEMGR (Time Manager)**: An early task organizer that allowed users to input schedules, assign priority weights (1–10), and view sorted chronological timetables.
*   **RESEARCHASST (Research Assistant)**: A cross-referencing bibliography tool designed for storing reference indices.
*   **Insta-Mail & Insta-Writer**: Mail-merge word processors designed to link name lists directly to form letters.

## 2. Technical Mechanics: Time Manager Priority Scheduling

To replicate the core technical logic of **TOTL.TIMEMGR** on-chain, we implement a **Task Priority Scheduler** in Yul. Tasks are stored in a packed memory block, sorted and fetched by priority.

### 2.1 Task Slot Layout (64-bit word)
To keep gas costs optimal, each scheduled task occupies 64 bits:
*   **Bits `[0..15]`**: Time tick (0–65535, representing time of day)
*   **Bits `[16..23]`**: Priority Weight (1 = Lowest, 10 = Highest)
*   **Bits `[24..63]`**: Task ID / Identifier tag (40 bits)

### 2.2 Yul Priority Sorter
The following Yul function iterates through a list of packed tasks and finds the highest-priority task that is ready to execute at the current time:

```yul
// Scans a list of packed tasks and returns the ID of the highest priority task ready at targetTime
// taskPtr: Start memory address of task array
// count: Number of tasks in the list
// targetTime: Current simulated clock tick
function getHighestPriorityTask(taskPtr, count, targetTime) -> bestTaskId {
    let bestPriority := 0
    bestTaskId := 0

    for { let i := 0 } lt(i, count) { i := add(i, 1) } {
        // Load 64-bit packed task word
        let offset := mul(i, 8)
        let word := mload(add(taskPtr, offset))
        
        let timeTick := and(word, 0xFFFF)
        let priority := and(shr(16, word), 0xFF)
        let taskId := and(shr(24, word), 0xFFFFFFFFFF) // Extract 40-bit ID

        // Check if task is ready (timeTick <= targetTime)
        let isReady := iszero(gt(timeTick, targetTime))

        if isReady {
            // Check if this task has higher priority than the best one found so far
            if gt(priority, bestPriority) {
                bestPriority := priority
                bestTaskId := taskId
            }
        }
    }
}
```
